# V5-Derived Quality Standard

## Core Standard

The article must help a buyer choose, calculate, compare, or prepare a useful project brief. It should make uncertainty easier to navigate instead of presenting several numbers as equally valid.

## Direct Answer

- Answer the main query within the first 120 words.
- Name one default recommendation when the evidence supports it.
- State who the default is for and what assumptions it uses.
- Separate a daily or normal recommendation from compact, occasional, maximum, or exceptional values.

Use this pattern when multiple ranges exist:

1. **Default:** the value most readers should select.
2. **Verified compact option:** use only after checking named constraints.
3. **Occasional maximum:** do not present as the everyday specification.
4. **Project exception:** defer to actual products, plans, standards, or qualified professionals.

## Units and Numbers

- Use one unit system throughout the article and tables.
- Default to inches for a United States or unspecified Shopify audience.
- Do not mix feet and inches in the same guide unless the user explicitly needs both; if both are required, make one primary and show conversions consistently.
- Recalculate every worked example after changing a recommendation.
- Check that capacity charts agree with the prose and FAQ.
- Label planning ranges as guidance rather than universal rules or code requirements.

## Structure

- One H1.
- Six to ten H2 sections.
- A direct-answer opening, decision framework, practical constraints, project-preparation section, CTA, and final FAQ.
- Exactly five FAQ questions.
- FAQ is the final H2; no conclusion, CTA, sources heading, or other content follows it.
- The fifth FAQ answer contains the source links so research remains at the end without breaking the FAQ-last rule.

## Evidence

- Use at least three credible sources for measurable or technical topics.
- Include at least one primary or authoritative source when available.
- Explain what each source supports in a short phrase.
- Make editorial recommendations distinguishable from quoted standards.
- Never convert competitor opinions into universal facts.

## Originality and Tone

- Do not copy headings, FAQ sets, sentences, examples, case studies, or CTA language from reference blogs.
- Avoid generic filler, fake quotations, invented stories, and unsupported superlatives.
- Prefer concrete verbs, buyer questions, formulas, checks, and tradeoffs.
- Keep the brand present where expertise changes the recommendation, not in every section.
- Use short paragraphs, scannable lists, and tables only when comparison benefits.

## Bilingual Parity

The Chinese review copy must preserve:

- heading order;
- numerical values and units;
- recommendation hierarchy;
- factual qualifications;
- links and source descriptions;
- CTA meaning;
- image placements and alt-text intent.

Treat a mismatch between languages as a factual error.

## Images

Plan at least five images and preferably seven for a long guide:

1. hero showing the full product or use case;
2. scale, dimensions, or seating context;
3. detail that supports a key decision;
4. working or real-use configuration;
5. design or material comparison;
6. installation, support, or access context when relevant;
7. closing image aligned with the CTA.

Every plan item requires a purpose, insertion point, filename, alt text, aspect ratio, and prompt. Generated images are illustrative and must not be described as customer projects.
